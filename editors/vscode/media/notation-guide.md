# 青空文庫記法 — 完全リファレンス

このプラグインで扱える「青空文庫テキスト記法 (Aozora notation)」のすべてを、
**書く順** にまとめました。困ったらここを開いてください
(`Ctrl+Shift+P` → `Aozora: 記法ガイドを開く`)。

---

## 0. ざっくり全体像

青空文庫は **プレーンテキスト** を「組版指示込みで書く」ためのプロジェクト。
拡張子は `.txt` ですが、プラグインは `.afm` / `.aozora` / `.aozora.txt` も
aozora モードで開きます。

5 種類の記号が登場します:

| 記号 | 役割 | 例 |
|---|---|---|
| `［＃...］` | **注記** (組版指示) | `［＃改ページ］` |
| `《...》` | **ルビ読み** | `青空《あおぞら》` |
| `｜` | **ルビベース指定** | `｜青空《あおぞら》` |
| `※` | **外字マーカー** | `※［＃「木＋吶のつくり」、第3水準1-85-54］` |
| `〔...〕` | **アクセント分解** | `〔Crevez chiens〕` |

すべて全角文字。半角の `[` `]` `<<` `|` を打つと、補完で全角に変換されます
(下記 §10 参照)。

---

## 1. ルビ (読みがな)

### 暗黙ルビ (kanji + 《reading》)

漢字の連続のあとに `《》` で読みを書くだけ:

```
青空《あおぞら》文庫を読む。
```

→ レンダリング: 青空(あおぞら)文庫を読む。

### 明示ルビ (｜base《reading》)

ベースが漢字以外、または曖昧な場合は `｜` でベースの開始を明示:

```
そんな｜事《こと》があったのか。
ABC《エービーシー》という会社で働く。
```

`｜` が無いと、`そんな事《こと》` は `そんな事` 全体ではなく
直前の漢字 1 文字 (事) だけにルビが掛かるかもしれず曖昧。
**迷ったら ｜ を付ける** のが原則。

---

## 2. 注記 (組版指示)

`［＃...］` の中に **キーワード** を書く。代表的なもの:

| キーワード | 効果 |
|---|---|
| `改ページ` | ページ区切り |
| `中央揃え` | 段落中央寄せ開始 |
| `ここから〇字下げ` / `ここで字下げ終わり` | インデント開始/終了 |
| `太字` / `太字終わり` | 強調 |
| `「target」に傍点` | 直前の `target` 文字列に傍点 |
| `本文中ここで段下げ` | 段の改行 |
| `ここから罫囲み` / `ここで罫囲み終わり` | 罫線囲い |

例:

```
本文の途中で［＃改ページ］次のページへ続く。

［＃中央揃え］はじまり［＃中央揃え終わり］

［＃ここから2字下げ］
インデント中の文章。
複数行にまたがる。
［＃ここで字下げ終わり］

これは［＃「重要」に傍点］な部分。
```

**ヒント**: `[#` (半角) を打つと候補が出てきて、選ぶと全角の
`［＃canonical］` に変換されます (下記 §10)。

---

## 3. 外字 (JIS X 0213 / Unicode)

辞書に無い字や、底本にしか無い字を **形の説明 + 参照番号** で書く:

```
※［＃「木＋吶のつくり」、第3水準1-85-54］
※［＃「キリル文字 Ya」、U+042F］
※［＃「ハート」、U+2665、12-3］
```

3 つの形式:

1. **JIS X 0213 mencode**: `第3水準1-85-54` / `第4水準2-1-1`
   面 (1 or 2) - 区 - 点 で字を一意に指定。
2. **Unicode 直書き**: `U+XXXX` (16 進、最大 6 桁)
3. **底本ページ-行**: `12-3` のような末尾 (情報のみ、解決には使われない)

プラグインは:
- mencode → 字 (LSP の inlay hint で `→ 枘` のように表示)
- Unicode → 字 (parse_u_plus でランタイム解決)
- description → 字 (8800 件のフォールバック辞書)

カーソルが `※[#...]` の上にある時は raw 表記、それ以外の時は **解決字 1 文字**
に折りたたまれて表示されます (`aozora.gaijiFold.enabled` で切替可)。

---

## 4. 鉤括弧 (会話・引用)

```
「お早う」と母が言った。
「で、どうしますか?」
```

特別な意味は持たず、普通の括弧。但し**注記の中で直前を target 指定**するときは:

```
そう｜呟《つぶや》いた［＃「呟」に傍点］。
```

---

## 5. アクセント分解 (欧文)

ASCII 範囲外のラテン文字 (フランス語アクサン、ドイツ語ウムラウト) を
ASCII で代替表記する:

```
〔Cre^ne!〕
〔Cre'pe〕
〔fu:r〕
```

| 記号 | 意味 |
|---|---|
| `'` (アポストロフィ) | アキュート (´) |
| `` ` `` (バッククォート) | グレーブ (`) |
| `^` | サーカムフレックス (^) |
| `~` | チルド (~) |
| `:` | ダイエレシス / ウムラウト (¨) |
| `&` | 上リング (˚) |
| `,` | セディラ (¸) |
| `_` | マクロン (¯) |
| `/` | ストローク |

---

## 6. 縦中横 (横書き数字をブロック化)

縦書きで「2 桁の数字を横並びに 1 文字ぶんの幅に収める」記法:

```
昭和［＃「12」は縦中横］年
```

## 7. 帰り点 (漢文)

```
学而［＃「（一）」］時習［＃「（一）レ」］之
```

## 8. ヘッダ (作品メタデータ)

ファイル冒頭の慣習的なヘッダ:

```
タイトル
作者
（訳者など）

-------------------------------------------------------
【テキスト中に現れる記号について】

《》：ルビ
｜：ルビの付く文字列の始まりを特定する記号
［＃］：入力者注

-------------------------------------------------------
[本文がここから始まる]
```

40 個以上の `-` で囲まれた領域はプリアンブル。プラグインの auto-detect が
「これは aozora 記法」と判断するシグナルでもあります。

## 10. プラグイン側のショートカット

### 半角→全角の emmet 補完

半角キーを 1 つ打つだけで全角の対応字に変換 + その場で空の括弧ペアに wrap し、
カーソルを中身に置く動作です。途中で考えを変えて何も入れずに Backspace すると、
空のペアごと自動で削除されます。

| 半角 | → | 全角 (＋ wrap) | 備考 |
|---|---|---|---|
| `[` | → | `［┃］` | カーソルは中央 |
| `]` | → | `］` | 単独で閉じ括弧を打ちたい時用 |
| `<` | → | `《reading┃》` | reading が選択状態、Tab で外へ |
| `>` | → | `》` | 単独で閉じる時用 |
| `\|` | → | `｜base《reading》┃` | base が選択、Tab で reading、Tab で外 |
| `*` | → | `※［＃「description┃」、mencode］` | 外字注記の雛形 |
| `{` | → | `〔┃〕` | アクセント分解 (kikkou) |
| `}` | → | `〕` | 単独で閉じる時用 |
| `#` | → | `［＃┃］` | 注記。ここでさらに候補が出て slug カタログ補完 |
| `[#` | → | `［＃canonical］` | カタログから直接 canonical slug を確定 |

### 範囲ラップ

選択して右クリ → **「Aozora: 選択を括弧で囲む」サブメニュー**:

- `《...》` ルビ
- `《《...》》` 二重ルビ
- `「...」` 鉤括弧
- `〔...〕` 亀甲括弧
- `［＃...］` 注記
- `｜...《》` ルビベース化
- `［＃「...」に傍点］` 傍点付加

キーバインド:
- `Ctrl+Alt+R` ルビで囲む
- `Ctrl+Alt+B` 傍点を付ける

### プレビュー

右クリ → **Aozora: Open Preview** で HTML プレビューを横に開きます。
ファイル変更に追従します。

### inlay hint

`※[#…]` の右に `→ X` の形で解決字が表示されます。
`gaijiFold` を無効にしても inlay は出るので、軽い確認用にどうぞ。

## 11. 診断 (エラー) コード一覧

LSP がエラーや警告を出すときに使う `code`:

| code | 重み | 意味 |
|---|---|---|
| `aozora::source-contains-pua` | warning | 私用領域文字 (U+E001..U+E004) が混入 |
| `aozora::unclosed-bracket` | error | 開き括弧が閉じていない |
| `aozora::unmatched-close` | error | 対応する開きの無い閉じ括弧 |
| `aozora::residual-annotation-marker` | warning | `［＃...］` が分類できなかった |

各コードはエディタの「Problems」ペインで確認、Quick Fix (Ctrl+.) で
自動修正候補が出ます。
